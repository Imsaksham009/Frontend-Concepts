# Types, Coercion, Copying, and Built-in Objects — The Complete Guide

---

## PART 1 — Types in JavaScript

JavaScript has two fundamental categories of types. Understanding this division explains almost every surprising behavior in the language.

---

### The Two Categories

**Primitive Types** — stored directly by value in the stack. There are exactly 7:

1. `number`
2. `string`
3. `boolean`
4. `null`
5. `undefined`
6. `symbol`
7. `bigint`

**Reference Types** — stored by reference, object lives in the heap:

1. `object` (plain objects, arrays, dates, regex, null — yes null)
2. `function` (technically an object with `[[Call]]`)

The distinction between these two categories drives everything — how variables are assigned, how equality works, how copying works, and how functions receive arguments.

---

### Primitive Types — Deep Dive

#### `number`

JavaScript has only **one** number type. There is no separate `int` or `float`. Everything is a 64-bit floating point number (IEEE 754 double precision).

```javascript
console.log(0.1 + 0.2);         // 0.30000000000000004 — floating point imprecision
console.log(0.1 + 0.2 === 0.3); // false!

// The safe way to compare floats
console.log(Math.abs(0.1 + 0.2 - 0.3) < Number.EPSILON); // true

// Special number values
console.log(Infinity);   // Infinity
console.log(-Infinity);  // -Infinity
console.log(NaN);        // NaN — Not a Number, but typeof NaN === 'number'!

// NaN is the only value in JS not equal to itself
console.log(NaN === NaN);       // false
console.log(NaN == NaN);        // false
console.log(Number.isNaN(NaN)); // true — the correct way to check

// Number limits
console.log(Number.MAX_SAFE_INTEGER); // 9007199254740991 (2^53 - 1)
console.log(Number.MIN_SAFE_INTEGER); // -9007199254740991

// Beyond safe integer range — precision is lost
console.log(9007199254740991 + 1); // 9007199254740992 — correct
console.log(9007199254740991 + 2); // 9007199254740992 — WRONG! same answer
```

#### `string`

Strings are **immutable** in JavaScript. Every string operation returns a new string — the original is never modified.

```javascript
let str = 'hello';
str[0] = 'H'; // silently does nothing
console.log(str); // 'hello' — unchanged

// Strings are immutable — methods return new strings
const upper = str.toUpperCase(); // 'HELLO' — new string
console.log(str);   // 'hello' — original unchanged
console.log(upper); // 'HELLO'

// String primitives vs String objects
const primitive = 'hello';
const object    = new String('hello');

console.log(typeof primitive); // 'string'
console.log(typeof object);    // 'object'

// They behave differently with equality
console.log(primitive == object);  // true — coercion happens
console.log(primitive === object); // false — different types!

// Never use new String() — always use string literals
```

#### `boolean`

Only two values: `true` and `false`. But the concept of **truthy** and **falsy** extends far beyond just these two values.

**The complete list of falsy values — memorize these:**

```javascript
// Every falsy value in JavaScript — exactly 8
false
0
-0
0n        // BigInt zero
''        // empty string
null
undefined
NaN

// Everything else is truthy — including these surprises:
console.log(Boolean([]));        // true — empty array is truthy!
console.log(Boolean({}));        // true — empty object is truthy!
console.log(Boolean('0'));       // true — string '0' is truthy!
console.log(Boolean('false'));   // true — string 'false' is truthy!
console.log(Boolean(-1));        // true — negative numbers are truthy!
console.log(Boolean(Infinity));  // true
```

This is critical because JavaScript uses truthiness/falsiness extensively in conditions:

```javascript
// All these conditions check truthiness
if (value) { }
value ? 'yes' : 'no'
value || 'default'
value && doSomething()
```

#### `null`

Represents the **intentional absence** of a value. It's something a developer explicitly sets to indicate "no value here".

```javascript
let user = null; // explicitly no user

// The famous typeof null bug
console.log(typeof null); // 'object' — this is a historical bug in JS
                           // It will NEVER be fixed (would break the web)

// Check for null explicitly
if (user === null) { } // correct
if (!user) { }         // works but also catches undefined, 0, '', etc.
```

#### `undefined`

Represents a variable that has been declared but not yet assigned a value. It's the language's way of saying "this doesn't have a value yet".

```javascript
let x;
console.log(x); // undefined — declared but not assigned

function greet(name) {
    console.log(name); // undefined if called without argument
}
greet(); // undefined

const obj = {};
console.log(obj.nonExistent); // undefined — property doesn't exist

// null vs undefined
console.log(null == undefined);  // true — they're loosely equal
console.log(null === undefined); // false — different types

// The practical distinction:
// null  = intentionally empty (you set it)
// undefined = accidentally empty (engine set it)
```

#### `symbol`

Symbols are **always unique** values, primarily used as object property keys to avoid name collisions.

```javascript
const id1 = Symbol('id');
const id2 = Symbol('id');

console.log(id1 === id2); // false — ALWAYS unique, even with same description

// Used as unique object keys
const ID = Symbol('id');
const user = {
    name: 'Peter',
    [ID]: 123 // symbol key — won't conflict with any string key
};

console.log(user[ID]);    // 123
console.log(user['id']);  // undefined — completely different key

// Symbols are non-enumerable by default
console.log(Object.keys(user));   // ['name'] — symbol not included
console.log(Object.getOwnPropertySymbols(user)); // [Symbol(id)]

// Global symbols — shared across the codebase
const globalId = Symbol.for('id');
const sameId   = Symbol.for('id');
console.log(globalId === sameId); // true — same symbol from global registry
```

#### `bigint`

For integers larger than `Number.MAX_SAFE_INTEGER`. Created by appending `n` to a number:

```javascript
const bigNumber = 9007199254740991n; // BigInt
const regular   = 9007199254740991;  // Number

// Operations stay precise beyond safe integer range
console.log(bigNumber + 1n); // 9007199254740992n — correct!
console.log(regular   + 1);  // 9007199254740992  — correct
console.log(regular   + 2);  // 9007199254740992  — WRONG (regular number)
console.log(bigNumber + 2n); // 9007199254740993n — correct!

// Cannot mix BigInt and Number
console.log(1n + 1);   // TypeError
console.log(1n + 1n);  // 2n — must use BigInt with BigInt

// typeof
console.log(typeof 42n); // 'bigint'
```

### Type Checking — All the Ways

```javascript
// typeof — works for primitives (except null)
typeof 42          // 'number'
typeof 'hello'     // 'string'
typeof true        // 'boolean'
typeof undefined   // 'undefined'
typeof Symbol()    // 'symbol'
typeof 42n         // 'bigint'
typeof function(){} // 'function'
typeof {}          // 'object'
typeof []          // 'object' — arrays are objects!
typeof null        // 'object' — the famous bug

// instanceof — checks prototype chain
[] instanceof Array    // true
[] instanceof Object   // true — arrays are objects
{} instanceof Object   // true

// Object.prototype.toString — most accurate type checking
Object.prototype.toString.call(42)          // '[object Number]'
Object.prototype.toString.call('hello')     // '[object String]'
Object.prototype.toString.call(true)        // '[object Boolean]'
Object.prototype.toString.call(null)        // '[object Null]'
Object.prototype.toString.call(undefined)   // '[object Undefined]'
Object.prototype.toString.call([])          // '[object Array]'

Object.prototype.toString.call(/regex/)     // '[object RegExp]'

// Specific checks
Array.isArray([]);      // true — best way to check for arrays
Number.isNaN(NaN);      // true — best way to check for NaN
Number.isFinite(42);    // true
Number.isInteger(42);   // true
Number.isInteger(42.5); // false
```

## PART 2 — Type Coercion

Type coercion is JavaScript's automatic conversion of values from one type to another. It's the source of JavaScript's most infamous quirks — and once you understand it completely, none of those quirks will ever surprise you again.

There are two kinds:

**Implicit coercion** — JavaScript converts automatically, without you asking.
**Explicit coercion** — You intentionally convert using built-in functions.

---

### Explicit Coercion — You're in Control

```javascript
// To Number
Number('42')        // 42
Number('3.14')      // 3.14
Number('')          // 0
Number('  ')        // 0
Number('hello')     // NaN
Number(true)        // 1
Number(false)       // 0
Number(null)        // 0
Number(undefined)   // NaN
Number([])          // 0
Number([3])         // 3
Number([1,2])       // NaN

// parseInt and parseFloat — more forgiving
parseInt('42px')    // 42 — stops at first non-numeric char
parseInt('3.9')     // 3 — integer only
parseFloat('3.14')  // 3.14
parseInt('hello')   // NaN
parseInt('0xFF', 16) // 255 — parse as hexadecimal

// To String
String(42)          // '42'
String(true)        // 'true'
String(false)       // 'false'
String(null)        // 'null'
String(undefined)   // 'undefined'
String([1,2,3])     // '1,2,3'
String({})          // '[object Object]'
(42).toString()     // '42'
(255).toString(16)  // 'ff' — hexadecimal

// To Boolean
Boolean(0)          // false
Boolean('')         // false
Boolean(null)       // false
Boolean(undefined)  // false
Boolean(NaN)        // false
Boolean(1)          // true
Boolean('hello')    // true
Boolean([])         // true — IMPORTANT
Boolean({})         // true — IMPORTANT

// Shorthand conversions
+'42'        // 42  — unary plus converts to number
+true        // 1
+false       // 0
+null        // 0
+undefined   // NaN
!!value      // converts to boolean
`${42}`      // '42' — template literal to string
```

---

### Implicit Coercion — Where the Quirks Live

#### The `+` Operator — String Concatenation vs Addition

The `+` operator has dual personality. It adds numbers, but if either operand is a string, it concatenates:

```javascript
// Number + Number = addition
1 + 2          // 3

// String + anything = concatenation
'1' + 2        // '12' — number coerced to string
1 + '2'        // '12' — number coerced to string
'1' + '2'      // '12'
'1' + true     // '1true'
'1' + null     // '1null'
'1' + undefined // '1undefined'
'1' + {}       // '1[object Object]'
'1' + []       // '1'

// The classic confusion
1 + 2 + '3'    // '33' — left to right: (1+2)+'3' = 3+'3' = '33'
'1' + 2 + 3    // '123' — left to right: '1'+2 = '12', '12'+3 = '123'

// Other arithmetic operators ALWAYS convert to number
'5' - 2        // 3 — string coerced to number
'5' * 2        // 10
'5' / 2        // 2.5
'5' - '2'      // 3
true + true    // 2
true + false   // 1
[] + []        // '' — both arrays become empty strings
[] + {}        // '[object Object]'
{} + []        // 0 — {} treated as empty block, +[] = 0
```

#### Comparison Coercion — `==` vs `===`

`===` (strict equality) — **never** coerces. Both value AND type must match:

```javascript
1 === 1       // true
1 === '1'     // false — different types, no coercion
null === null // true
null === undefined // false
NaN === NaN   // false — NaN is never equal to anything
```

`==` (loose equality) — **coerces** types before comparing. The rules are complex:

```javascript
// null and undefined are only equal to each other
null == undefined  // true
null == 0          // false
null == ''         // false
null == false      // false

// Number vs String — string converted to number
1 == '1'           // true  — '1' → 1
0 == ''            // true  — '' → 0
0 == '0'           // true  — '0' → 0
'' == '0'          // false — no coercion between two strings

// Boolean vs anything — boolean converted to number first
true  == 1         // true  — true → 1
false == 0         // true  — false → 0
true  == '1'       // true  — true → 1, '1' → 1
false == ''        // true  — false → 0, '' → 0
false == null      // false — null only equals undefined
true  == 'true'    // false — true → 1, 'true' → NaN

// Object vs primitive — object converted via valueOf/toString
[1] == 1           // true  — [1].valueOf() → [1], toString() → '1' → 1
[] == false        // true  — [] → '' → 0, false → 0
[] == 0            // true
[0] == false       // true
```

The famous equality table quirks all come from these coercion rules. The practical takeaway:  **always use `===` in real code** . The only legitimate use of `==` is `value == null` which catches both `null` and `undefined` in one check.

```javascript
// The one acceptable use of ==
if (value == null) {
    // true for both null AND undefined
    // equivalent to: value === null || value === undefined
}
```

## PART 3 — Shallow Copy vs Deep Copy

This is one of the most practically important things to understand for real-world JavaScript.

---

### Why Copying Is Complicated

The problem exists only for **reference types** (objects and arrays). Primitives are always copied by value, so there's no issue:

```javascript
let a = 5;
let b = a; // copy of the VALUE 5
b = 10;
console.log(a); // 5 — completely unaffected
```

But with objects:

```javascript
const original = { name: 'Peter', address: { city: 'NYC' } };
const reference = original; // NOT a copy — just another reference to the same object

reference.name = 'Tony';
console.log(original.name); // 'Tony' — modifying reference modified original!
```

`reference` and `original` point to the  **same object in heap memory** . There is only one object. Both variables are just pointers to it.

---

### Shallow Copy — One Level Deep

A shallow copy creates a  **new top-level object** , but nested objects are still shared references:

```javascript
const original = {
    name: 'Peter',
    age: 25,
    address: { city: 'NYC', zip: '10001' }, // nested object
    hobbies: ['coding', 'reading']           // nested array
};
```

#### Method 1 — Spread Operator `{...obj}`

```javascript
const shallow = { ...original };

// Top level properties are independent
shallow.name = 'Tony';
console.log(original.name); // 'Peter' — unaffected ✅

// But nested objects are still shared!
shallow.address.city = 'LA';
console.log(original.address.city); // 'LA' — AFFECTED! ❌

shallow.hobbies.push('gaming');
console.log(original.hobbies); // ['coding', 'reading', 'gaming'] — AFFECTED! ❌
```

#### Method 2 — `Object.assign({}, obj)`

```javascript
const shallow = Object.assign({}, original);

// Same behavior as spread — shallow only
shallow.name = 'Tony';          // doesn't affect original ✅
shallow.address.city = 'LA';    // DOES affect original ❌
```

#### Method 3 — Array spread and `slice` for arrays

```javascript
const arr = [1, 2, [3, 4], { name: 'Peter' }];

const shallowArr1 = [...arr];
const shallowArr2 = arr.slice();
const shallowArr3 = Array.from(arr);

// All three behave the same — shallow copy
shallowArr1[0] = 99;
console.log(arr[0]); // 1 — unaffected ✅

shallowArr1[2].push(5);
console.log(arr[2]); // [3, 4, 5] — AFFECTED! ❌
```

#### The Shallow Copy Mental Model

```
original ──────────────────────────────────────────→ { 
                                                       name: 'Peter',    ← primitive, copied by value
                                                       age: 25,          ← primitive, copied by value
                                                       address: ref1 ──→ { city: 'NYC' }  ← SHARED
                                                       hobbies: ref2 ──→ ['coding']       ← SHARED
                                                     }
                                                            ↑ ↑
shallow ────────────────────────────────────────────→ {    | |
                                                       name: 'Peter',    ← own copy
                                                       age: 25,          ← own copy
                                                       address: ref1 ────┘ same reference!
                                                       hobbies: ref2 ──────┘ same reference!
                                                     }
```

---

### Deep Copy — All Levels Independent

A deep copy creates a completely independent object — no shared references at any level.

#### Method 1 — `JSON.parse(JSON.stringify(obj))`

The quick and dirty approach. Works for simple data but has serious limitations:

```javascript
const original = {
    name: 'Peter',
    address: { city: 'NYC' },
    hobbies: ['coding', 'reading']
};

const deep = JSON.parse(JSON.stringify(original));

deep.address.city = 'LA';
console.log(original.address.city); // 'NYC' — completely independent ✅

deep.hobbies.push('gaming');
console.log(original.hobbies); // ['coding', 'reading'] — unaffected ✅
```

**Limitations — things it CANNOT handle:**

```javascript
const problematic = {
    fn: function() {},        // ❌ functions are lost — becomes undefined
    sym: Symbol('id'),        // ❌ symbols are lost
    undef: undefined,         // ❌ undefined is lost
    date: new Date(),         // ❌ Date becomes string, not Date object
    regex: /pattern/g,        // ❌ RegExp becomes empty object {}
    inf: Infinity,            // ❌ becomes null
    nan: NaN,                 // ❌ becomes null
    map: new Map(),           // ❌ becomes empty object {}
    set: new Set(),           // ❌ becomes empty object {}
};

const broken = JSON.parse(JSON.stringify(problematic));
console.log(broken.fn);   // undefined
console.log(broken.date); // string, not Date
console.log(broken.inf);  // null

// ALSO fails silently with circular references
const circular = { name: 'Peter' };
circular.self = circular;
JSON.stringify(circular); // TypeError: Converting circular structure to JSON
```

#### Method 2 — `structuredClone()` — The Modern Way

Introduced in 2022, now available in all modern browsers and Node.js 17+. This is the best built-in deep clone method:

```javascript
const original = {
    name: 'Peter',
    address: { city: 'NYC' },
    hobbies: ['coding'],
    date: new Date(),
    map: new Map([['key', 'value']]),
    set: new Set([1, 2, 3])
};

const deep = structuredClone(original);

deep.address.city = 'LA';
console.log(original.address.city); // 'NYC' ✅

// Handles types JSON can't
console.log(deep.date instanceof Date); // true ✅
console.log(deep.map instanceof Map);   // true ✅
console.log(deep.set instanceof Set);   // true ✅

// Also handles circular references!
const circular = { name: 'Peter' };
circular.self = circular;
const clonedCircular = structuredClone(circular); // works! ✅
```

**Limitations of `structuredClone`:**

```javascript
const withFunctions = {
    fn: () => 'hello',  // ❌ functions still cannot be cloned
    sym: Symbol('id'),  // ❌ symbols still cannot be cloned
};

structuredClone(withFunctions); // DataCloneError
```

#### Method 3 — Recursive Custom Deep Clone

When you need full control — handles everything including functions and symbols. This is exactly what your PDF implements:

```javascript
function deepClone(value, visited = new Map()) {
    // Handle primitives and null
    if (value === null || typeof value !== 'object') {
        return value;
    }
  
    // Handle circular references
    if (visited.has(value)) {
        return visited.get(value);
    }
  
    // Handle Date
    if (value instanceof Date) {
        return new Date(value.getTime());
    }
  
    // Handle RegExp
    if (value instanceof RegExp) {
        return new RegExp(value.source, value.flags);
    }
  
    // Handle Map
    if (value instanceof Map) {
        const mapClone = new Map();
        visited.set(value, mapClone);
        value.forEach((val, key) => {
            mapClone.set(deepClone(key, visited), deepClone(val, visited));
        });
        return mapClone;
    }
  
    // Handle Set
    if (value instanceof Set) {
        const setClone = new Set();
        visited.set(value, setClone);
        value.forEach(val => {
            setClone.add(deepClone(val, visited));
        });
        return setClone;
    }
  
    // Handle Array and Object
    const clone = Array.isArray(value) ? [] : {};
    visited.set(value, clone);
  
    // Handle regular string/number keys
    for (let key in value) {
        if (value.hasOwnProperty(key)) {
            clone[key] = deepClone(value[key], visited);
        }
    }
  
    // Handle Symbol keys
    Object.getOwnPropertySymbols(value).forEach(sym => {
        clone[sym] = deepClone(value[sym], visited);
    });
  
    return clone;
}

// Test it
const original = {
    name: 'Peter',
    address: { city: 'NYC', coords: { lat: 40.7, lng: -74.0 } },
    hobbies: ['coding', ['reading', 'writing']],
    date: new Date(),
    map: new Map([['key', { nested: true }]]),
    set: new Set([1, 2, { id: 3 }]),
};

// Circular reference
original.self = original;

const cloned = deepClone(original);

cloned.address.city = 'LA';
cloned.hobbies[1].push('gaming');

console.log(original.address.city);  // 'NYC' ✅
console.log(original.hobbies[1]);    // ['reading', 'writing'] ✅
```

---

### Comparison Table

| Method                  | Nested Objects | Functions | Date      | Map/Set  | Circular Refs | Speed   |
| ----------------------- | -------------- | --------- | --------- | -------- | ------------- | ------- |
| Spread `{...}`        | ❌ shallow     | ✅        | ✅        | ✅       | ✅            | Fastest |
| `Object.assign`       | ❌ shallow     | ✅        | ✅        | ✅       | ✅            | Fastest |
| `JSON`parse/stringify | ✅             | ❌ lost   | ❌ string | ❌`{}` | ❌ throws     | Fast    |
| `structuredClone`     | ✅             | ❌ throws | ✅        | ✅       | ✅            | Fast    |
| Custom recursive        | ✅             | ✅        | ✅        | ✅       | ✅            | Slowest |

**Rule of thumb:**

* Simple nested data, no special types → `structuredClone`
* Need to copy functions/symbols → custom recursive clone
* Performance critical, flat object → spread or `Object.assign`
* Legacy code → `JSON.parse(JSON.stringify())`

## PART 4 — Built-in Objects

---

### `Object` — Static Methods You Must Know

```javascript
const person = {
    name: 'Peter',
    age: 25,
    city: 'NYC'
};

// Keys, values, entries
Object.keys(person);    // ['name', 'age', 'city']
Object.values(person);  // ['Peter', 25, 'NYC']
Object.entries(person); // [['name', 'Peter'], ['age', 25], ['city', 'NYC']]

// Reconstruct from entries
const fromEntries = Object.fromEntries([
    ['name', 'Peter'],
    ['age', 25]
]);
// { name: 'Peter', age: 25 }

// Useful pattern — transform object values
const prices = { apple: 1.5, banana: 0.5, cherry: 3.0 };
const doubled = Object.fromEntries(
    Object.entries(prices).map(([key, val]) => [key, val * 2])
);
// { apple: 3, banana: 1, cherry: 6 }

// Merge objects
const merged = Object.assign({}, obj1, obj2, obj3);
const merged2 = { ...obj1, ...obj2, obj3 }; // modern way

// Freeze — prevent all modifications
const frozen = Object.freeze({ name: 'Peter', age: 25 });
frozen.name = 'Tony';    // silently fails (or throws in strict mode)
frozen.job = 'Dev';      // silently fails
delete frozen.age;       // silently fails
console.log(frozen);     // { name: 'Peter', age: 25 } — unchanged

// Freeze is SHALLOW — nested objects still mutable
const obj = Object.freeze({ nested: { value: 1 } });
obj.nested.value = 99;   // works! nested object not frozen
console.log(obj.nested.value); // 99

// Seal — allow modifying existing, prevent adding/deleting
const sealed = Object.seal({ name: 'Peter', age: 25 });
sealed.name = 'Tony';    // ✅ modifying existing — allowed
sealed.job = 'Dev';      // ❌ adding new — fails silently
delete sealed.age;       // ❌ deleting — fails silently
console.log(sealed);     // { name: 'Tony', age: 25 }

// Check state
Object.isFrozen(frozen); // true
Object.isSealed(sealed); // true
```

### `Map` — Key-Value Pairs with Any Key Type

`Map` is like an object but with crucial differences. It's one of the most useful modern data structures.

**Key differences from plain objects:**

| Feature          | Object                               | Map                    |
| ---------------- | ------------------------------------ | ---------------------- |
| Key types        | only strings/symbols                 | ANY type               |
| Order guaranteed | no (mostly yes in practice)          | yes — insertion order |
| Size             | manual (`Object.keys().length`)    | `.size`property      |
| Iteration        | indirect                             | direct iteration       |
| Performance      | slower for frequent add/delete       | optimized              |
| Default keys     | has inherited keys (`toString`etc) | completely empty       |

```javascript
// Creating a Map
const map = new Map();

// Setting values — ANY type as key
map.set('string', 'string key');
map.set(42, 'number key');
map.set(true, 'boolean key');

const objKey = { id: 1 };
map.set(objKey, 'object key'); // objects as keys!

const fnKey = () => {};
map.set(fnKey, 'function key'); // functions as keys!

// Getting values
map.get('string');  // 'string key'
map.get(42);        // 'number key'
map.get(objKey);    // 'object key' — same reference needed
map.get({ id: 1 }); // undefined — different object reference!

// Checking existence
map.has('string');  // true
map.has('missing'); // false

// Size
map.size; // 5

// Deleting
map.delete('string'); // true — deleted
map.delete('missing'); // false — wasn't there

// Clearing all
map.clear();

// Iterating — in insertion order
const scores = new Map([
    ['Peter', 95],
    ['Tony',  88],
    ['Steve', 92]
]);

// iterate entries
for (const [name, score] of scores) {
    console.log(`${name}: ${score}`);
}
// Peter: 95, Tony: 88, Steve: 92

// Keys, values, entries
[...scores.keys()];    // ['Peter', 'Tony', 'Steve']
[...scores.values()];  // [95, 88, 92]
[...scores.entries()]; // [['Peter', 95], ['Tony', 88], ['Steve', 92]]

scores.forEach((score, name) => {
    console.log(`${name}: ${score}`);
});

// Convert to/from object
const obj = Object.fromEntries(scores);
// { Peter: 95, Tony: 88, Steve: 92 }

const backToMap = new Map(Object.entries(obj));
```

**When to use Map over Object:**

```javascript
// Use Map when:
// 1. Keys are not strings/symbols
const elementData = new Map();
const domElement = document.getElementById('btn');
elementData.set(domElement, { clicks: 0, created: Date.now() });

// 2. You frequently add and delete properties
const cache = new Map(); // better performance than object for this

// 3. You need insertion order guaranteed
const orderedConfig = new Map([
    ['first', 1],
    ['second', 2],
    ['third', 3]
]);

// 4. You need to know the size easily
console.log(orderedConfig.size); // 3 — direct, no calculation needed

// Use plain object when:
// 1. Keys are strings and structure is fixed
// 2. Working with JSON (Map doesn't serialize to JSON)
// 3. Accessing properties frequently with dot notation
```

### `Set` — Unique Values Collection

A `Set` stores **unique values** of any type. Duplicate additions are silently ignored.

```javascript
// Creating a Set
const set = new Set([1, 2, 3, 2, 1]); // duplicates removed
console.log(set); // Set {1, 2, 3}

// Adding values
set.add(4);
set.add(2); // already exists — ignored silently
set.add('hello');
set.add({ id: 1 }); // objects by reference — this IS added (new reference)
set.add({ id: 1 }); // THIS IS ALSO added — different reference!

// Checking
set.has(1);       // true
set.has(99);      // false

// Size
set.size; // number of unique values

// Deleting
set.delete(1); // true
set.delete(99); // false

// Iterating — in insertion order
const fruits = new Set(['apple', 'banana', 'cherry']);

for (const fruit of fruits) {
    console.log(fruit);
}

[...fruits.values()]; // ['apple', 'banana', 'cherry']

fruits.forEach(fruit => console.log(fruit));

// The most common use case — removing duplicates from array
const withDuplicates = [1, 2, 3, 2, 1, 4, 3, 5];
const unique = [...new Set(withDuplicates)];
// [1, 2, 3, 4, 5]

// Set operations
const setA = new Set([1, 2, 3, 4]);
const setB = new Set([3, 4, 5, 6]);

// Union — all elements from both
const union = new Set([...setA, ...setB]);
// {1, 2, 3, 4, 5, 6}

// Intersection — only in both
const intersection = new Set([...setA].filter(x => setB.has(x)));
// {3, 4}

// Difference — in A but not B
const difference = new Set([...setA].filter(x => !setB.has(x)));
// {1, 2}
```

### `WeakMap` — Map With Weak References

`WeakMap` is like `Map` but with two critical differences:

1. **Keys must be objects** (no primitives)
2. **Keys are held weakly** — if the key object has no other references, it can be garbage collected. The WeakMap entry disappears automatically.

```javascript
let obj1 = { name: 'Peter' };
let obj2 = { name: 'Tony' };

const weakMap = new WeakMap();
weakMap.set(obj1, 'some data');
weakMap.set(obj2, 'other data');

weakMap.get(obj1); // 'some data'
weakMap.has(obj1); // true

// When obj1 is no longer referenced anywhere else
obj1 = null; // the { name: 'Peter' } object becomes eligible for GC
             // The WeakMap entry for it will automatically be removed
             // No memory leak!

// WeakMap is NOT iterable — no .keys(), .values(), .forEach()
// No .size property
// These limitations exist because GC can remove entries at any time
```

**The perfect use case — private data for objects:**

```javascript
// Pattern: associate private data with objects without memory leaks
const privateData = new WeakMap();

class Person {
    constructor(name, age, ssn) {
        // Public data on the instance
        this.name = name;
      
        // Private data in WeakMap — not accessible from outside
        privateData.set(this, { age, ssn });
    }
  
    getAge() {
        return privateData.get(this).age;
    }
  
    // ssn never exposed
    validateSSN(input) {
        return privateData.get(this).ssn === input;
    }
}

const peter = new Person('Peter', 25, '123-45-6789');
console.log(peter.name);        // 'Peter' — public
console.log(peter.age);         // undefined — not on instance
console.log(peter.getAge());    // 25 — through method
peter.validateSSN('123-45-6789'); // true

// When peter is garbage collected, WeakMap entry is too
// No memory leak even though privateData is a global variable
```

**Another perfect use case — caching without memory leaks:**

```javascript
const cache = new WeakMap();

function processElement(domElement) {
    if (cache.has(domElement)) {
        return cache.get(domElement); // return cached result
    }
  
    const result = expensiveOperation(domElement);
    cache.set(domElement, result);
    return result;
}

// When domElement is removed from the DOM and dereferenced,
// the cache entry is automatically cleaned up
// With a regular Map, you'd have a memory leak
```

### `WeakSet` — Set With Weak References

Like `WeakMap` but for unique values. Values must be objects. Values are held weakly.

```javascript
const weakSet = new WeakSet();

let obj = { name: 'Peter' };
weakSet.add(obj);
weakSet.has(obj); // true

obj = null; // object eligible for GC, weakSet entry disappears

// Main use case — tracking objects without preventing GC
const processed = new WeakSet();

function process(obj) {
    if (processed.has(obj)) {
        console.log('already processed');
        return;
    }
  
    // do processing...
    processed.add(obj);
}

// Common use — marking DOM nodes
const clickedElements = new WeakSet();

document.querySelectorAll('button').forEach(btn => {
    btn.addEventListener('click', function() {
        if (clickedElements.has(this)) return; // already handled
        clickedElements.add(this);
        handleFirstClick(this);
    });
});
// When buttons are removed from DOM, WeakSet entries clean up automatically
```

### `Map` vs `WeakMap` vs `Object` — When to Use What

```
Need string/symbol keys, JSON serialization, dot notation access?
→ Use plain Object {}

Need any key type, size tracking, ordered iteration, frequent mutations?
→ Use Map

Need object keys, want automatic GC when keys are dereferenced,
associating data with objects (DOM nodes, class instances)?
→ Use WeakMap

Need unique values, fast lookup, deduplication?
→ Use Set

Need unique object references, GC-friendly tracking?
→ Use WeakSet
```

## The Complete Mental Model

**Types** divide into primitives (stored by value on the stack, immutable, compared by value) and reference types (stored by reference on the heap, compared by reference). This single division explains assignment behavior, equality checks, and copying behavior.

**Coercion** is JavaScript's automatic type conversion — implicit coercion follows rules that are consistent but complex, especially around `+` and `==`. Knowing the rules turns "JavaScript is weird" into "JavaScript is predictable".

**Copying** has two levels — shallow (new top-level container, shared nested references) and deep (fully independent at all levels). Use `structuredClone` for modern deep cloning, spread/assign for shallow.

**Built-in objects** like `Map`, `Set`, `WeakMap`, and `WeakSet` were added because plain objects and arrays couldn't handle every use case efficiently. Choosing the right data structure — not just defaulting to `{}` and `[]` for everything — is a mark of mature JavaScript development.
